Performance & Scaling Layer

Critical services to implement before scaling beyond 1,000 creators. Every tool below has a specific, quantifiable reason.

⚡
Redis — Caching & Rate Limiting
Upstash (serverless) · or Self-hosted on DO
PRIORITY #1

Without caching, every visitor to a creator's storefront triggers a fresh Supabase query. At 10,000 concurrent visitors hitting multiple stores, your Supabase connection pool (limited to 60 concurrent connections on the free tier, 200 on Pro) will be exhausted within seconds, throwing 500 errors to buyers.

The Math

A typical store metadata query takes ~120ms from Supabase (India region). A Redis in-memory read takes ~1ms. That's a 120× speed improvement per request, and zero Postgres connection used.

Implementation strategy: Cache store metadata (title, colors, logo, product list) with a 60-second TTL. On store update, bust the specific cache key. Use Upstash (pay-per-request, no cold-starts) until you hit $20/month, then self-host Redis on a $6 DO droplet.

Rate limiting: Use Redis sliding-window counters to block IPs making more than 100 requests in 60 seconds. This blocks card-testing bots before they hit Cashfree's API.

Image Optimization & Delivery
ImageKit.io · Cloudflare Images · Cloudflare R2
PRIORITY #2

A creator uploads a 10MB raw photo from their iPhone as a store banner. Without optimization, every buyer downloads 10MB of pixels — on a 4G connection in Tier-2 India (average speed ~15Mbps), that's a 5+ second load just for the hero image. Google's Core Web Vitals will penalize this store in search rankings.

The Impact

Google research shows a 1-second delay in mobile load time reduces conversions by 20%. On Indian mobile networks, unoptimized images are the single largest bottleneck. ImageKit converts to WebP automatically, reducing file size by 60–80%.

ImageKit setup: Use ImageKit as your origin storage URL transformer. Point it at your Supabase Storage bucket. All image URLs become ik.imagekit.io/digione/[path]?tr=w-800,f-webp — dynamically resized and compressed at the CDN edge.

Cloudflare Images ($5/month for 100K images) is a simpler alternative that integrates directly with Cloudflare's network if you're already using them for DNS.

Next.js ISR — Incremental Static Regeneration
Built into Next.js 15 · Zero additional cost
HIGH IMPACT

Creator storefronts don't change every second — yet if you use Server-Side Rendering (SSR), Next.js calls your database on every single page load. With ISR, Next.js generates the storefront HTML once, caches it at Cloudflare's edge, and only regenerates it when a creator updates their store.

Static HTML vs Dynamic SSR

A static HTML page served from Cloudflare's CDN can handle millions of concurrent requests for free. A dynamic SSR route uses a Vercel Serverless Function invocation (which costs money and has concurrency limits) every single time.

Implementation: Use export const revalidate = 60 in your storefront page.tsx. This tells Next.js to serve the cached HTML for 60 seconds, then regenerate it in the background. Use revalidatePath('/store/[slug]') in your store-update API route for instant cache busting on publish.

On-Demand ISR (Next.js 13+): When a creator hits "Publish" in their dashboard, call the revalidation webhook immediately — no waiting for the 60-second window. Buyers see changes instantly, database load stays near zero.

Application Monitoring
Sentry · OpenTelemetry · Betterstack
RELIABILITY

In production, you're flying blind without monitoring. A Cashfree payment webhook may start failing silently at 2 AM — you'll only discover it when creators complain that buyers paid but didn't receive their products. Sentry catches this in real-time and pages you before a single creator notices.

What to Monitor for DigiOne

Payment webhook success rate, storefront page load TTFB, file download generation time, API route P95 latency, and Supabase connection pool usage. Set Slack alerts when any metric breaches thresholds.

Sentry for Next.js auto-instruments all API routes with error tracking and performance tracing. The free tier includes 5,000 errors/month — more than enough for early-stage. Configure tracesSampleRate: 0.1 to sample 10% of transactions (reduces noise and volume).

Betterstack Uptime (free tier) pings your storefront URLs every 30 seconds from Mumbai and alerts you if load time exceeds 3s or the site goes down — critical because Cashfree and Indian buyers have low tolerance for downtime.

Background Jobs & Webhook Processing
Inngest · Trigger.dev · QStash (Upstash)
STABILITY

Vercel Serverless Functions have a strict 10-second timeout on the free tier (60 seconds on Pro). If a buyer purchases a product and you try to: verify the Cashfree webhook → write to Supabase → generate a PDF → send an email → post a WhatsApp message — all inside one API route, it will timeout and the buyer gets a broken experience.

The Cashfree Webhook Problem

Cashfree expects your webhook endpoint to respond with a 200 status within 5 seconds. If you do too much processing inside the webhook handler, it'll timeout, Cashfree will mark it failed and retry, causing duplicate deliveries. Use a job queue to immediately acknowledge the webhook and process async.

Inngest pattern for DigiOne: Your Cashfree webhook handler does ONE thing — writes the raw event to Inngest's queue and responds 200 immediately. Inngest then fans out to child jobs: send-confirmation-email, generate-download-link, notify-creator, update-analytics. Each runs independently with automatic retries.

QStash by Upstash is a simpler alternative — an HTTP-based message queue. Publish a message to a URL, and Upstash delivers it with retries and delays. Perfect if you don't need complex orchestration.

Dedicated Search Engine
Typesense (self-hosted) · Algolia · Meilisearch
CONVERSION

PostgreSQL's full-text search with pg_trgm is serviceable for small catalogs, but as creators list 50–200 products, ILIKE '%query%' queries become full table scans. They're also unforgiving — a typo like "Excell template" returns zero results instead of matching "Excel template."

Impact on Revenue

If a buyer searches and finds nothing due to a typo or slow results, they leave. A dedicated search engine with typo tolerance and sub-50ms results directly translates to higher conversion rates on creator stores.

Typesense is the recommended choice — it's open-source, can be self-hosted on a $6 DO droplet, and delivers sub-20ms typo-tolerant search. Its Node.js client integrates easily with Next.js API routes. Mirror your Supabase products table to Typesense using a Supabase webhook trigger on INSERT/UPDATE.

Meilisearch is another strong open-source option with a beautiful search-as-you-type API. Algolia is the most powerful but expensive ($1/1000 search requests beyond the free tier).

Infrastructure · 03

Security Hardening

Non-negotiable before accepting real payments. These protect your creators, their buyers, and your platform's reputation.

🤖
Bot Protection
CRITICAL
Cloudflare Turnstile (free, unlimited) should be placed on your Login, Signup, Checkout, and Contact pages. Unlike hCaptcha or reCAPTCHA, Turnstile runs silently in the background — buyers see no challenge puzzle, just a smooth experience.

Carding attacks are the #1 threat to Indian e-commerce platforms. Bots test thousands of stolen cards in rapid succession to find valid ones. Turnstile blocks this entirely before a single request reaches Cashfree. Cashfree may also suspend your merchant account if they detect carding activity — protect yourself.
🖼️
Content Moderation
HIGH
Sightengine or AWS Rekognition should scan every image upload before it's saved to Supabase Storage. A bad actor uploading NSFW or illegal content to a DigiOne store could get your entire digione.ai domain blacklisted by Cloudflare, Cashfree, or Google Safe Browsing — catastrophic for all creators on the platform.

Sightengine's free tier covers 2,000 checks/month. Call it synchronously on upload and reject with a clear error message. Log rejected uploads for manual review.
💾
Database Backups (PITR)
CRITICAL
Point-in-Time Recovery is your last line of defence against catastrophic data loss. A bug, a SQL migration gone wrong, or an accidental DELETE FROM users without a WHERE clause could wipe your entire business in seconds.

On Supabase Pro ($25/mo): Enable PITR in the dashboard — you can rewind to any second in the last 7 days. On self-hosted: Run pg_dump every 6 hours via a cron job, upload to a private Cloudflare R2 bucket, and test restoration monthly. Store at least 30 days of backups.
🔑
API Rate Limiting
HIGH
Use @upstash/ratelimit with Redis to implement sliding-window rate limits on every public API route. A scraper hitting your products API at 100 req/sec would dump your entire creator catalog within minutes — competitive intelligence theft and server load combined.

Recommended limits: 20 req/min for unauthenticated routes, 100 req/min for authenticated creators, 5 req/min for checkout endpoints. Return 429 Too Many Requests with a Retry-After header for legitimate clients.
🔒
Webhook Signature Verification
HIGH
Every Cashfree webhook must be verified using the HMAC-SHA256 signature they send in the x-webhook-signature header. A fraudster can POST a fake "payment successful" event to your webhook endpoint and receive a digital product without paying.

Verify the signature on every single webhook call before trusting the payload. This is a 5-line implementation that prevents complete revenue fraud. Never skip this step.
🌐
Security Headers & CSP
MEDIUM
Configure HTTP security headers in your next.config.ts: Content-Security-Policy (blocks XSS attacks), X-Frame-Options: DENY (prevents clickjacking), Strict-Transport-Security (forces HTTPS), and Permissions-Policy (restricts camera/mic access).

Use next-safe or @next/headers to generate your CSP. Also enable Supabase Row Level Security (RLS) policies on every table so creators can only access their own data — even if an API route has a bug, the database enforces access control.

Roadmap · 04

Scaling Roadmap

A phased approach to growing from MVP to millions of buyers — without rewriting everything.

1
Phase 1 · Current State
Vercel Free + Supabase Free — The MVP Stage

This is your current setup and it's appropriate for the MVP. Vercel's free tier gives you 100GB bandwidth/month, and Supabase's free tier gives you a 500MB Postgres database with 2GB file storage. This is sufficient for launching, onboarding your first creators, and validating the business model.

What you can realistically handle: ~1,000 active creators, 50,000 monthly storefront visitors across all stores, 500MB of creator-uploaded assets, and up to 50 concurrent database connections.

⚠ Chokepoints — Why It Will Fail
Vercel will throttle or lock your account the moment you exceed 100GB bandwidth in a month — a single viral creator could trigger this.
Supabase free tier limits database connections to ~20 simultaneous. A traffic spike from multiple stores will cause timeout cascades.
Supabase pauses your database after 7 days of inactivity (free tier) — unacceptable for a live product.
No background job processing means Cashfree webhooks run inside the API route — any slowness causes missed deliveries.
2
Phase 2 · Growth Stage (~$500–$2K MRR)
Upgrade the Foundations — Redis, ISR, Monitoring, Jobs

At this stage, you're processing real creator payments. The engineering priority shifts from "ship features fast" to "nothing breaks at 2 AM." Upgrade Supabase to Pro ($25/month) for PITR, connection pooling via PgBouncer, and no pausing. Add Vercel Pro ($20/month) for 1TB bandwidth and 60-second function timeouts.

→ Implement These in Order
Add Upstash Redis for store metadata caching and API rate limiting — immediate impact on database load and bot protection.
Migrate storefront routes to ISR with on-demand revalidation. Static pages served from Cloudflare CDN handle unlimited traffic for free.
Set up Inngest for Cashfree webhook processing, email delivery, and download link generation. Eliminates timeout failures.
Deploy Sentry with Slack alerting for API route errors and payment webhook failures. Know about issues before creators do.
Add Cloudflare Turnstile to Login, Signup, and Checkout pages to block carding bots.

Monthly infrastructure cost at this stage: Supabase Pro $25 + Vercel Pro $20 + Upstash ~$5 + Sentry Free + Inngest Free tier = ~$50/month. A creator platform making $500 MRR can easily cover this.

3
Phase 3 · Scale Stage ($2K+ MRR)
Migrate to DigitalOcean + Coolify + Cloudflare CDN

Vercel's Pro plan charges $0.40 per GB beyond 1TB of bandwidth, and managed Supabase limits you to 8GB database storage on Pro. When you have 10,000+ creators uploading gigabytes of content, these costs compound fast. The migration to a self-hosted setup is a one-time investment that cuts your infrastructure bill by 80%.

→ Migration Playbook
Spin up a DO Droplet ($48/month — 4 vCPU, 8GB RAM): This single server handles what Vercel charges $200+/month for. Install Coolify (a self-hosted PaaS, similar to Vercel UI) to manage deployments via Git push.
Self-host Supabase on the same droplet (or a separate $24/month droplet for DB isolation): You get unlimited storage and the full Supabase stack (Auth, Storage, Realtime, Edge Functions) for free.
Point Cloudflare CDN in front of your droplet: Cloudflare caches static assets and CDN pages. Your DigitalOcean server only processes cache misses. Bandwidth costs drop to near zero.
Use Cloudflare R2 for file storage (zero egress fees) instead of Supabase Storage or AWS S3. At scale, this alone saves hundreds of dollars monthly.
Run automated pg_dump backups to R2 every 6 hours using a cron job on the server. Retain 30 days of backups for disaster recovery.

Market Context · 06

India-Specific Considerations

Unique technical and product decisions driven by the Indian creator economy.

4G/5G

Mobile-First Performance

Over 96% of Indian internet users access the web via mobile. Average 4G speeds in Tier-2/3 cities hover around 10–20 Mbps. Every storefront must score 90+ on Google Lighthouse Mobile. Use ISR + CDN to ensure sub-1s load times even on mid-range Android devices.

UPI

UPI & Payment Diversity

UPI accounts for 80%+ of digital payments in India. Your Cashfree integration must prominently feature PhonePe, Google Pay, and Paytm UPI flows at checkout — not just cards. Also support Net Banking and Wallets for older demographics who don't trust UPI for large amounts.

95%

WhatsApp Open Rate

Indian buyers respond to WhatsApp more than email. Sending a digital product download link via WhatsApp within 30 seconds of purchase (via Interakt or Wati) dramatically reduces "I didn't receive my product" support tickets — the #1 complaint in Indian digital commerce.

GST

GST & Compliance (TDS)

Creators earning above ₹2.5L/year must file GST on digital products. When DigiOne handles creator payouts, you'll need to deduct TDS (Tax Deducted at Source) at 1% for creators with PAN and 5% without. Build this into your payout logic from Day 1 or face compliance issues.

20+

Multi-Language Support

India has 20+ official languages. Top creator categories — finance, fitness, cooking, education — have massive audiences in Hindi, Tamil, Telugu, and Bengali. Offering creators the ability to set their storefront language (with right-to-left support for Urdu) unlocks Tier-2/3 city markets.

INR

Pricing & Purchasing Power

Indian buyers are highly price-sensitive. A digital course priced at $97 (₹8,100) that converts well in the US might not convert in India. Support INR pricing, let creators offer EMI options (Cashfree supports this via credit card EMIs), and build a "Pay What You Want" feature for freemium products.

Architecture at a Glance — Request Flow

A buyer visits a creator's DigiOne store → Cloudflare CDN serves the cached static HTML (0 DB queries) → if cache miss, Next.js ISR on Vercel/DO regenerates from Redis → Redis miss hits Supabase Postgres (pooled via PgBouncer) → buyer checks out via Cashfree → Cashfree fires webhook to /api/cashfree/webhook → route verifies HMAC signature, enqueues job in Inngest, returns 200 in <100ms → Inngest fans out: sends confirmation via Resend, fires WhatsApp via Interakt, logs to Supabase, updates Redis cache → Sentry monitors every step, alerts Slack on failure.
