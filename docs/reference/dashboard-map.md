### Editor packages

**`src/components/dashboard/site-edit/editor/`** (2026-06-18 premium redesign — linkinbio; 2026-06-19 generalized to single-page):
`EditorShell.tsx` (the generic premium chrome: sidebar nav · dotted canvas · preview column; nav/section-meta/`siteType`/preview injected as props; floating brand Save with dirty state). `LinkInBioShell.tsx` is now a thin wrapper over `EditorShell` supplying the link-in-bio nav + iPhone `PreviewPane`; the single-page page composes `EditorShell` directly. **`src/hooks/site-editor/`** (2026-06-20 — shared editor logic extracted from the four edit pages, which previously hand-rolled each one):
- `useEditorHistory.ts` — undo/redo **and** dirty tracking in one hook (`build`/`apply`/`deps`/`enabled`). A single effect schedules the debounced snapshot and marks dirty, and short-circuits during a restore — so undo/redo never spuriously marks dirty and there's no cross-effect flag-ordering bug. Returns `{ canUndo, canRedo, undo, redo, dirty, setDirty }`. Snapshot deps cover the content fields; slug/publish/settings are marked dirty by a tiny per-editor effect (they aren't in the snapshot).
- `useUnsavedChanges.ts` — `pendingNav`/`guardedNavigate`/`cancel`/`discardAndLeave`/`saveAndLeave` + the `beforeunload` prompt. Pairs with `UnsavedChangesDialog`.
- `useSlugCheck.ts` — debounced `/api/sites/check-slug`; regex mirrors the server (`^[a-z0-9][a-z0-9-]{1,48}[a-z0-9]$`, 3–50). Derives idle/invalid/checking synchronously, only the remote verdict is state.
